# Проверка прав на материалы

Дата проверки: `2026-05-04`

## Что проверено

Проведена первичная инвентаризация материалов проекта по данным репозитория:

- состав папок `assets/`;
- карта источников и документы по наполнению слайдов;
- пакет для издательства;
- финальный файл `PPTX`;
- скрипт сборки `src/build-final-presentation.ps1`.

Проверка носит организационный характер и фиксирует наличие или отсутствие подтверждений внутри проекта. Это не юридическое заключение.

## Ключевой вывод

На текущий момент в репозитории **нет отдельного реестра прав, лицензий, разрешений или актов передачи материалов**.

Поэтому:

- можно уверенно описать, **какие материалы используются**;
- можно разделить их по уровню риска;
- **нельзя подтвердить по содержимому репозитория**, что все материалы разрешены к внешней передаче и публикации без ограничений.

## Предварительная оценка по группам

### Группа A. Условно контролируемые материалы

Сюда можно отнести материалы, которые с высокой вероятностью принадлежат заказчику или были подготовлены внутри проекта:

- `assets/бренд-материалы/логотип велесстрой.png`
- часть `assets/исходные-фото`, где по названию видно связь с площадкой или сотрудником компании
- часть `assets/working-visuals`, если они были нарисованы или собраны внутри проекта с нуля

Статус:

- использовать можно только после подтверждения, что заказчик действительно передал эти материалы для данного проекта;
- для логотипа желательно отдельное подтверждение права использования в учебной презентации и при передаче издательству.

### Группа B. Материалы с неподтвержденным происхождением

Сюда попадает большая часть рабочих визуалов:

- `assets/working-visuals/22-36`
- `assets/working-visuals/37-59`
- `assets/working-visuals/60-68`
- `assets/working-visuals/69-76`
- часть файлов `77-95`

Проблема:

- в репозитории нет файла-реестра, где было бы указано, кто автор каждого визуала;
- нет указания, какие изображения созданы заново, какие взяты из старых презентаций, какие из фотоархива, а какие получены из внешних источников;
- нет лицензий, ссылок на покупку, актов передачи или письменных разрешений.

Статус:

- эту группу нельзя считать очищенной по правам.

### Группа C. Старые презентации и пособия

Сюда относятся:

- `assets/исходные-презентации/КУРС для СТРОПАЛЬЩИКА_III.pptx`
- `assets/исходные-презентации/КУРС для СТРОПАЛЬЩИКА_ПРАКТИКА.pptx`
- `assets/исходные-презентации/КУРС для СТРОПАЛЬЩИКА_III.pdf`
- `assets/исходные-пособия/ПОСОБИЕ ДЛЯ СТРОПАЛЬЩИКА.pdf`
- сканы в `assets/исходные-пособия/Стропальные работы пособие/`
- `assets/исходные-пособия/Итоговые тесты_Стропальщик_Русский.docx`

Риск:

- это содержательные источники, но в репозитории нет подтверждения, кому принадлежат права;
- часть финальной презентации напрямую основана на `КУРС для СТРОПАЛЬЩИКА_III.pptx`;
- сканы пособия и тестовый документ могут быть как внутренними материалами заказчика, так и сторонними учебными материалами.

Статус:

- использовать как источник смысла возможно внутри проекта;
- для внешней передачи, тиражирования, переработки или публикации нужны подтверждения происхождения и условий использования.

## Что отсутствует в проекте

Внутри репозитория не найдено:

- отдельного файла `LICENSE`, `COPYING` или аналогичного реестра прав в `assets/`;
- таблицы соответствия `файл -> правообладатель -> основание использования`;
- ссылок на договор, письмо-согласование, служебную записку или лицензию;
- маркировки `собственное фото`, `материал заказчика`, `купленный сток`, `перерисовано нами`, `генерация`.

## Что можно считать установленным уже сейчас

- финальный `PPTX` зависит от старой презентации `КУРС для СТРОПАЛЬЩИКА_III.pptx`;
- поздний блок `77-95` использует собственные рабочие визуалы и тестовый документ;
- значительная часть материалов находится в серой зоне с точки зрения подтверждения прав не потому, что права точно отсутствуют, а потому, что **в проекте нет подтверждающих данных**.

## Что нужно запросить, чтобы закрыть проверку

Минимальный набор подтверждений:

1. Подтверждение от заказчика, что он передал права или разрешение на использование:
- старых презентаций;
- учебного пособия и тестового файла;
- бренд-материалов;
- исходных фото с площадки и сотрудников.

2. Таблица по `working-visuals`:
- из какого источника взят каждый файл;
- был ли он создан заново;
- использовалась ли генерация;
- можно ли передавать его издательству и использовать в финальной презентации.

3. Отдельное решение по слайдам `1-76`:
- можно ли оставлять их на базе `КУРС для СТРОПАЛЬЩИКА_III.pptx` без переработки;
- либо нужно заменить спорные визуалы на собственные или подтвержденные.
