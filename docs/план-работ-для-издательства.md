# План работ для издательства

## Назначение документа

Этот документ используется для управления проектом подготовки презентации издательством на основе утвержденного технического задания.

Документ нужен для:

- поэтапного планирования работ;
- распределения задач между заказчиком и издательством;
- контроля сроков и готовности материалов;
- фиксации контрольных точек и результатов каждого этапа.

## Моя роль в проекте

В рамках этого проекта моя роль заключается не в самостоятельном изготовлении итоговой презентации, а в помощи заказчику при постановке задачи издательству и сопровождении проекта.

Моя функция:

- перевести техническое задание в понятный производственный план;
- разбить проект на этапы, задачи и контрольные точки;
- помочь подготовить комплект материалов для передачи в издательство;
- помочь контролировать полноту, качество и логичность результата на каждом этапе;
- помочь проверить, что итоговая презентация соответствует ТЗ.

## Основной принцип работы

Работа строится по схеме:

`Техническое задание -> план работ -> передача материалов в издательство -> контроль промежуточных результатов -> приемка итоговой версии`

## Этапы проекта

### Этап 1. Подготовка проекта к передаче в издательство

Цель этапа: собрать и оформить все исходные данные, чтобы издательство могло начать работу без разночтений.

- [ ] Проверить актуальность технического задания.
- [ ] Убедиться, что в ТЗ зафиксированы цели, аудитория, объем, формат и сроки.
- [ ] Проверить, что в ТЗ отражены требования к нумерации слайдов, брендированной плашке и заголовку раздела в верхней части слайда.
- [ ] Сформировать перечень исходных материалов, которые будут переданы издательству.
- [ ] Подготовить старые презентации для использования как вспомогательный материал.
- [ ] Подготовить отсканированное учебное пособие.
- [ ] Подготовить фотографии, пригодные для использования.
- [ ] Подготовить логотипы, фирменные цвета, шрифты и иные материалы фирменного стиля.
- [ ] Подготовить образец оформления плашки с номером слайда.
- [ ] Подготовить аватар и иные утвержденные визуальные материалы.
- [ ] Проверить, какие материалы можно использовать без ограничений по правам.
- [ ] Определить, какие изображения потребуется дополнительно подобрать или сгенерировать.

Результат этапа:

- утвержденное ТЗ;
- собранный комплект исходных материалов;
- готовый пакет для передачи в издательство.

### Этап 2. Постановка задачи издательству

Цель этапа: передать издательству полную и однозначную задачу.

- [ ] Передать издательству техническое задание.
- [ ] Передать издательству план работ и ожидаемые контрольные точки.
- [ ] Передать комплект исходных материалов.
- [ ] Отдельно объяснить, что презентация должна быть учебной, аттестационной и рассчитанной на длительное использование.
- [ ] Отдельно обозначить, что основная аудитория - новички и часть слушателей может слабо понимать русский язык.
- [ ] Отдельно зафиксировать требование: больше визуальной информации, меньше текста на слайдах.
- [ ] Отдельно зафиксировать требование: на каждом слайде логотип компании.
- [ ] Отдельно зафиксировать требование: название раздела в верхней части каждого слайда.
- [ ] Отдельно зафиксировать требование: номер слайда в нижнем правом углу на брендированной плашке.
- [ ] Согласовать с издательством состав промежуточных результатов.
- [ ] Согласовать сроки предоставления черновой, промежуточной и финальной версий.

Результат этапа:

- издательство получило полный комплект материалов;
- зафиксированы сроки и ожидаемые результаты по этапам.

### Этап 3. Разработка структуры презентации

Цель этапа: создать и согласовать логику презентации до начала полной дизайнерской сборки.

- [ ] Подготовить послайдовый или пораздельный план презентации.
- [ ] Проверить, что структура укладывается в целевой объем.
- [ ] Проверить, что основные учебные блоки расположены в правильной последовательности.
- [ ] Проверить, что в начале есть вводный блок по допуску к работе.
- [ ] Проверить, что подробно раскрыт алгоритм выполнения работ.
- [ ] Проверить, что отдельно раскрыты безопасность и практические навыки.
- [ ] Проверить, что подробно раскрыта знаковая сигнализация.
- [ ] Проверить, что отражены голосовая связь, радиосвязь и правило единого языка команд.
- [ ] Проверить, что отражены взаимодействие, иерархия и распределение ролей.
- [ ] Проверить, что включены темы браковки стропов, тары, крюков и грузозахватных приспособлений.
- [ ] Проверить, что включены действия при аварийной ситуации и при выявлении нарушений.
- [ ] Проверить, что по темам добавлены наглядные примеры ошибок или инцидентов.
- [ ] Проверить, что в конце предусмотрены критерии аттестации, инструкция по заполнению бланка и 15 тестовых вопросов.
- [ ] Утвердить структуру до начала детального дизайна.

Результат этапа:

- согласованная структура презентации;
- понятный каркас всех будущих слайдов.

### Этап 4. Разработка текстового и методического содержания

Цель этапа: подготовить содержание, на основе которого издательство сможет собирать слайды.

- [ ] Подготовить краткие тексты для каждого раздела.
- [ ] Подготовить тезисы для каждого слайда или группы слайдов.
- [ ] Подготовить текст докладчика как учебный материал для тренировки.
- [ ] Проверить, что формулировки понятны аудитории начального уровня.
- [ ] Проверить, что сложные формулировки упрощены без потери смысла.
- [ ] Проверить, что терминология единообразна по всей презентации.
- [ ] Проверить, что законодательная база указана кратко и корректно.
- [ ] Проверить, что акцент сделан на практике, а не на перегрузке нормативными цитатами.
- [ ] Подготовить или передать тестовые вопросы.
- [ ] Проверить, что каждый тестовый вопрос имеет только один правильный ответ.

Результат этапа:

- методически готовое содержание для сборки презентации;
- подготовленный текст докладчика;
- готовый тестовый блок.

### Этап 5. Разработка визуальной концепции издательством

Цель этапа: согласовать общий внешний вид презентации до массовой верстки всех слайдов.

- [ ] Получить от издательства 2-3 образца ключевых слайдов.
- [ ] Проверить соответствие фирменному стилю компании.
- [ ] Проверить, что логотип размещен корректно и единообразно.
- [ ] Проверить, что верхний заголовок раздела читается и не перегружает слайд.
- [ ] Проверить, что номер слайда расположен в нижнем правом углу на брендированной плашке.
- [ ] Проверить, что цветовая маркировка понятна и используется последовательно.
- [ ] Проверить, что текст достаточно крупный для показа группе 15-20 человек.
- [ ] Проверить, что на слайдах нет перегрузки мелким текстом.
- [ ] Проверить, что визуальный стиль подходит для учебной и производственной тематики.
- [ ] Проверить, что иллюстрации читаемы и помогают обучению.
- [ ] Утвердить визуальную концепцию до верстки всей презентации.

Результат этапа:

- утвержденный визуальный шаблон;
- понятные правила оформления всех слайдов.

### Этап 6. Производство презентации издательством

Цель этапа: собрать полную версию презентации на основе согласованной структуры и визуальной концепции.

- [ ] Издательство собирает полный комплект учебных слайдов.
- [ ] Издательство добавляет критерии аттестации.
- [ ] Издательство добавляет слайд с инструкцией по заполнению бланка.
- [ ] Издательство добавляет 15 тестовых слайдов.
- [ ] Проверить полноту учебных разделов.
- [ ] Проверить соответствие количества слайдов целевому объему.
- [ ] Проверить единообразие оформления во всей презентации.
- [ ] Проверить наличие верхнего названия раздела на каждом слайде.
- [ ] Проверить наличие номера слайда на каждом слайде.
- [ ] Проверить корректность расположения логотипов.
- [ ] Проверить качество изображений, схем, иконок и инфографики.
- [ ] Проверить отсутствие водяных знаков на используемых иллюстрациях.

Результат этапа:

- первая полная версия презентации.

### Этап 7. Проверка и доработка

Цель этапа: убедиться, что готовая версия соответствует ТЗ и учебным задачам.

- [ ] Проверить соответствие презентации техническому заданию.
- [ ] Проверить, что содержание логично и легко воспринимается.
- [ ] Проверить, что ключевые практические действия объяснены наглядно.
- [ ] Проверить, что темы безопасности раскрыты достаточно подробно.
- [ ] Проверить, что примеры ошибок и инцидентов уместны и понятны.
- [ ] Проверить, что тестовый блок соответствует учебной части.
- [ ] Проверить, что презентация подходит для доклада продолжительностью около 1 часа 20 минут.
- [ ] Подготовить список замечаний для издательства.
- [ ] Передать замечания издательству.
- [ ] Получить исправленную версию.
- [ ] Проверить устранение замечаний.

Результат этапа:

- согласованная исправленная версия презентации.

### Этап 8. Финальная приемка

Цель этапа: принять итоговый результат и зафиксировать завершение работ.

- [ ] Проверить финальную версию `PPTX`.
- [ ] Проверить, что все правки внесены.
- [ ] Проверить, что структура, содержание и оформление соответствуют ТЗ.
- [ ] Проверить, что нумерация слайдов полная и последовательная.
- [ ] Проверить, что брендированная плашка с номером оформлена корректно.
- [ ] Проверить, что на каждом слайде указан раздел в верхней части.
- [ ] Проверить, что файл открывается без ошибок.
- [ ] Проверить, что шрифты, изображения и оформление не съезжают при открытии.
- [ ] Принять итоговую версию.
- [ ] Архивировать исходные материалы и рабочие версии проекта.

Результат этапа:

- принятая итоговая презентация;
- закрытый проект.
